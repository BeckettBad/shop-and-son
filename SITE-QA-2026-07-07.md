## Security findings

1. SANITIZER GAPS (the one real code item, low-medium). `sanitizeShopifyHtml`
   (`src/lib/storefront-client.ts:153-172`) is a denylist feeding `innerHTML` in
   `product-view.ts:695` and `policies.astro:65`. Bypasses: tab/newline-obfuscated
   `java\tscript:` URLs survive the `startsWith("javascript:")` check; SVG `xlink:href` is not
   checked; `<base>`/`<meta>` tags are not stripped. Requires malicious Shopify-side content
   plus a user click; no session to steal on a static site. Fix: allowlist sanitization
   (DOMPurify or equivalent) or extend the strip list. Good Codex brief item.
2. NO CSP. GitHub Pages cannot set headers and no `<meta http-equiv="Content-Security-Policy">`
   exists. Adding one to `Base.astro` is the only available mitigation and would blunt any
   residual XSS. Second Codex brief item.
3. DEPENDENCIES. `npm audit` production: 2 low (esbuild dev-server, Windows-only, not in built
   output). Dev-only: 4 moderate in the `@astrojs/check` -> `yaml` chain (linter DoS, zero
   shipped impact). Astro 6.4.6 -> 7.0.6 clears the esbuild advisory; breaking upgrade,
   schedule deliberately.
4. CI. `deploy.yml` is least-privilege, injection-free, first-party actions only. Nice-to-have:
   pin actions to SHAs.
5. CREDENTIALS: fully clean. Admin keys exist nowhere on disk (empty placeholders in
   `.env.example` only). `homepage/.env` never committed (verified across all 272 commits, all
   refs). Built output contains only the public read-only Storefront token, by design. One
   action: confirm in Shopify admin that the token's scopes are read-only.
6. HOUSEKEEPING. `WebSA.md` (business strategy doc) sits untracked at the root of this public
   repo - move it to the Studio vault before it gets committed by accident. No robots.txt or
   sitemap ships. `public/preorders/images/fam-tatoo.jpg` is untracked - commit or remove.

Verified clean: XSS sinks otherwise all `textContent`/parameterized GraphQL; URL params
allowlisted or encoded; no open redirects; no eval/document.write; no third-party scripts
